기본 문법-태그



1)태그란?
:HTML과 같은 언어를 만든 사람들은 이 언어를 만들 때
자신들이 만든 문법을 사람들이 쉽게 이해하기를 원했음.
그래서 일상에서 자주 쓰는 말 중에 이 문법과 비슷한 말을 찾았음.
그래서 찾은 비유,은유가 옷을 살 때 붙어 있는 것이 태그.
태그가 하는 역할이 무엇인가? > 그 옷을 설명함. 그래서 태그임.

2)태그의 특징
-시작과 종료 태그가 존재함
-열린 태그 = "/"가 없음, 닫힌 태그 = "/"가 있음
-태그는 "태그" 내용 "/태그"으로 사용

-태그 중첩이 가능함
-태그에는 부모 자식 관계 같은 고정된 관계인 태그들이 있음



3)자주 사용되는 태그
  • head: 메타 데이터의 집합으로 웹 페이지에 직접적으로 보이지 않는 정보임.
    • head태그는 title, meta, link, style, script 태그로 구성됨.
  • body: 본문의 내용을 담는 공간임.
  • html: html 파일임을 정의. html 태그는 head 태그와 body 태그로 이루어짐.
  • title: title 태그는 페이지를 대표하는 내용의 제목을 넣어야 함.
  • meta: meta 태그는 으로 스스토 닫는 태그이기 때문에 이렇게 나타낸다.
  • div:
  • a
  • script
  • link
  • img
  • p
  • span
  • li
  • ul
  • br
  • style
  • h1~h6
  • input
  • form
  • strong

    • 4)태그 속성

  • 태그 속성은 태그의 이름만으로 정보를 다 표현하기 부족하다고
    느낀 사람들이 만든 것으로 속성이라는 문법을 추가해
    정보를 좀 더 풍부하게 표현할 수 있게 해준다.
    • ex)<태그이름 속성="값">


    참고)W3C(The World Wide Web Consortium)
    :국제민간표준화기구의 홈페이지
    웹은 한 기업에 의해서 만들어지는 것이 아니고 W3C에 소속된 여러 기업과 기관들의 논의를 통해서 만들어지고 있습니다.
    미래에는 어떤 기능이 필요하고, 그 기능은 어떤 태그로 표현할 것인가를 의논하는 것이죠.
    W3C에서 웹의 미래가 결정되면 각각의 브라우저 업체들이 같은 태그를 보다 잘 구현하기 위해서 치열하게 경쟁하고 있습니다.
    웹이 다른 기술들과 구분되는 공공재적인 면모라고 할 수 있습니다.
